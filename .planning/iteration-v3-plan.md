## 개선 목표

### 목표 1: rev2-why.md EXECUTE 섹션 실습형으로 전환 (N1)
- 대상 파일: `references/rev2-why.md`
- 현재 상태: EXECUTE 섹션이 "다음을 생각해보세요:" + 사고 질문 2개로만 구성됨. 다른 레볼루션의 EXECUTE는 모두 Claude에게 직접 요청하는 실습형 지시문("Claude에게 ~라고 요청해보세요", "~를 실행해보세요")인 반면, rev2만 유일하게 "생각해보세요" 형태의 사고 실험으로 되어 있음. SKILL.md Phase A 프로토콜("참조 문서의 EXECUTE 섹션을 읽고 실습 안내")과 불일치
- 목표 상태: 기존 사고 질문을 유지하면서, Claude에게 직접 요청하는 실습 항목을 추가. 예: "Claude에게 '지금 쓰는 업무 도구의 한계를 CLI로 해결할 방법을 제안해줘'라고 요청해보세요", "PPTX 수작업 과정을 Claude에게 설명하고 자동화 방법을 물어보세요" 등
- 우선순위: Major (EXECUTE 섹션의 패턴 일관성. 비개발자가 "생각만 하고 끝나는" 레볼루션이 됨)

### 목표 2: rev3→rev4 순서 전환 연결 문구 추가 (N2)
- 대상 파일: `prompts/core-features.md` (마지막 줄)
- 현재 상태: core-features.md 162행에 "→ 레볼루션 4로 이동 안내."로만 끝남. 레볼루션 3은 MCP, Subagent, Agent Teams, Hook, Plugin 등 고급 기능을 다루고, 레볼루션 4는 CLI 기초(cd, ls, pwd), Git 기초(commit, push) 등 기본 개념을 다룸. 고급→기초 순서가 의도적 설계(체험 우선 학습)임에도 불구하고 학습자에게 "왜 갑자기 기초로 돌아가는지" 설명하는 연결 문구가 전혀 없음
- 목표 상태: rev3 종료 시 "고급 기능을 먼저 체험했으니, 이제 기초 도구를 정리합니다. 이미 알고 있다면 건너뛰어도 됩니다."와 같은 맥락 전환 안내 문구 추가. basics.md 도입부에 이미 "이미 알고 있다면 건너뛸 수 있습니다"가 있으므로, core-features.md 종료 시 연결 문구만 보강하면 충분
- 우선순위: Minor (학습 흐름의 자연스러움 향상. 기능 결함은 아니나, 학습자 혼란 방지)

### 목표 3: rev1-experience.md 데모 1 예시를 비개발자 친화적으로 보강 (N3)
- 대상 파일: `references/rev1-experience.md`
- 현재 상태: 데모 1에서 Skill 예시로 `/weekly-sync`를 사용 ("Slack 메시지 수집 + git 로그 분석 + 보고서 생성"). 이 예시는 Slack, git 로그 등 개발자 도구를 전제로 하여 비개발자 대상 학습자에게 와닿지 않을 수 있음. 레볼루션 1은 "첫 체험"이므로 학습자의 즉각적 공감이 중요
- 목표 상태: `/weekly-sync` 예시를 유지하되, 비개발자도 공감할 수 있는 부연 설명 또는 대체 예시를 병기. 예: "팀 업무 보고서를 한 명령어로 자동 생성" 또는 PPTX 프로젝트 맥락에 맞춘 "/pptx-maker" 같은 예시 추가. SKILL.md에서 이미 PPTX 연결로 "PPTX 만들어줘" 체험을 명시하고 있으므로, 참조 파일에서도 이 맥락을 반영
- 우선순위: Minor (학습 효과 개선. 기능 결함은 아니나, 첫인상에 영향)

### 목표 4: rev3-3-mcp.md npx/JSON 코드 블록에 비개발자용 안내 보강 (N4)
- 대상 파일: `references/rev3-3-mcp.md`
- 현재 상태: 설정 방법 섹션에 `.mcp.json` JSON 코드 블록과 `npx` 명령어가 등장(38~49행). Node.js 안내 주석(51~53행)이 이미 존재하나, JSON 형식 자체("왜 이렇게 생겼는지", "이것을 직접 편집해야 하는지")에 대한 비개발자용 설명이 부족. PPTX 적용 예시(61~69행)에서도 두 번째 JSON 코드 블록이 추가 설명 없이 나옴
- 목표 상태: JSON 코드 블록 앞에 "이 설정은 Claude에게 '이런 도구를 사용할 수 있다'고 알려주는 설명서와 같습니다. 직접 편집할 필요 없이 Claude에게 '~MCP 서버를 추가해줘'라고 요청하면 됩니다."와 같은 맥락 설명 1~2줄 추가. 코드를 이해하지 못해도 개념을 파악할 수 있도록 안내
- 우선순위: Minor (기술적 장벽 완화. Node.js 안내는 이미 있으므로, JSON 형식에 대한 보완적 설명)

### 목표 5: rev3-6-hook.md 보안 경고 위치 및 강조 수준 점검 (N5)
- 대상 파일: `references/rev3-6-hook.md`
- 현재 상태: 보안 경고가 99행에 위치 (`> ⚠️ **보안 주의**: Hook은 **셸 명령어를 직접 실행**합니다. 신뢰할 수 없는 코드나 외부 입력을 Hook 명령어에 그대로 포함하지 마세요.`). 이 경고는 PPTX 프로젝트 적용 예시 코드 블록 하단(PostToolUse + Stop Hook JSON 이후)에 인라인 주석으로 포함되어 있음. v1에서 보안 경고를 추가한 이력이 있으나, 현재 위치가 예시 코드 아래 blockquote 안이라 눈에 잘 띄지 않을 수 있음
- 목표 상태: 보안 경고를 유지하되, EXPLAIN 섹션의 "핵심 원리" 직후(더 이른 위치)에도 간단한 보안 참고 문구를 추가하여, 학습자가 코드 예시에 도달하기 전에 보안 인식을 갖도록 함. 기존 99행 경고는 그대로 유지하여 이중 안내 구조
- 우선순위: Minor (보안 인식 강화. 기존 경고가 존재하므로 위치 보완 수준)
