## 이슈별 수정 우선순위 및 지침

### N1 (우선순위: Major)
- **현재 상태**: rev2-why.md 50~54행의 EXECUTE 섹션이 "다음을 생각해보세요:" + 사고 질문 2개로만 구성. 전체 레볼루션 중 유일한 think-only 패턴. SKILL.md Phase A 프로토콜 불일치.
- **수정 지침**:
  - rev2-why.md 52~54행을 다음으로 교체:
  ```
  다음을 직접 시도해보세요:
  1. Claude에게 "지금 쓰는 업무 도구의 한계를 CLI로 해결할 방법을 제안해줘"라고 요청해보세요
  2. PPTX를 수작업으로 만드는 과정을 Claude에게 설명하고, "이 과정을 자동화할 방법을 알려줘"라고 물어보세요
  3. (생각해보기) 지금 사용하는 도구 중 "버튼이 없어서 못 하는 것"이 있나요?
  ```
  - 기존 사고 질문 1개를 "(생각해보기)" 형태로 유지, 실습 항목 2개를 앞에 추가
  - 사고 질문 2번("터미널에서 AI가 자유롭게 작업한다면...")은 실습 1~2로 자연스럽게 체험되므로 통합

### 점수 산출 근거 상세

**구조 완성도 9→9.5**: N1 해소로 전 레볼루션의 EXECUTE 섹션이 "실습형 지시문" 패턴으로 통일됨. 현재 rev2만 사고 실험 형태인 것이 유일한 구조적 불일치였으므로 이를 해소하면 완전 통일.

**내용 일관성 8→8.5**: N2 해소로 rev3→rev4 전환 시 학습자에게 맥락을 제공. 커리큘럼 순서의 의도적 설계를 명시적으로 안내하여, 레볼루션 간 연결 논리가 강화됨.

**비개발자 접근성 8→9**: N3(비개발자 예시 부연), N4(JSON 코드 앞 맥락 설명), N5(보안 경고 조기 배치) 세 건 모두 비개발자의 학습 경험을 직접 개선. 특히 N4는 기술적 장벽이 가장 높았던 MCP 섹션의 접근성을 크게 향상시킴.
