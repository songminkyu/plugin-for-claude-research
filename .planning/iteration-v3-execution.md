
- rev1 EXECUTE(47행): "다음을 직접 시도해보세요:" + 관찰/실행 항목 4개
- rev3-3 EXECUTE(86행): "Claude에게 ~라고 요청해보세요" 형태 3개
- rev3-6 EXECUTE(107행): "Claude에게 ~라고 요청해보세요" 형태 3개
- **rev2 EXECUTE(52행): "다음을 생각해보세요:" + 사고 질문 2개** (유일한 think-only)

**권장 수정**: 52~54행을 다음과 같이 변경:

```markdown
## EXECUTE

다음을 직접 시도해보세요:
1. Claude에게 "지금 쓰는 업무 도구의 한계를 CLI로 해결할 방법을 제안해줘"라고 요청해보세요
2. PPTX를 수작업으로 만드는 과정을 Claude에게 설명하고, "이 과정을 자동화할 방법을 알려줘"라고 물어보세요
3. (생각해보기) 지금 사용하는 도구 중 "버튼이 없어서 못 하는 것"이 있나요?
```

기존 사고 질문 1개를 유지하면서, Claude에게 직접 요청하는 실습 항목 2개를 앞에 추가. 사고 질문 2번("터미널에서 AI가 자유롭게 작업한다면...")은 실습 1~2로 자연스럽게 체험하게 되므로 통합.

## N2 점검: rev3→rev4 순서 전환 연결 문구

**현재 상태**: core-features.md 162행 (파일 마지막 줄)

```markdown
→ 레볼루션 4로 이동 안내.
```

**문제**: 레볼루션 3은 MCP, Subagent, Agent Teams, Hook, Plugin 등 고급 기능을 다루고, 레볼루션 4는 CLI 기초(cd, ls, pwd), Git 기초(commit, push) 등 기본 개념을 다룸. 고급→기초 순서가 의도적 설계(체험 우선 학습)임에도 불구하고, 학습자에게 "왜 갑자기 기초로 돌아가는지" 설명하는 연결 문구가 없음. 단순히 "→ 레볼루션 4로 이동 안내."로만 끝남.

**권장 수정**: 162행을 다음과 같이 변경:

```markdown
→ 레볼루션 3에서 고급 기능을 먼저 체험했습니다. 레볼루션 4에서는 이 기능들의 기반이 되는 기초 도구(터미널, Git 등)를 정리합니다. 이미 알고 있는 내용이라면 건너뛸 수 있습니다.
→ 레볼루션 4로 이동 안내.
```

기존 "→ 레볼루션 4로 이동 안내." 앞에 맥락 전환 안내 1줄을 추가. basics.md 도입부에 이미 "이미 알고 있다면 건너뛸 수 있습니다"가 있으므로, 이를 미리 언급하여 학습자 불안을 해소.

## N3 점검: rev1-experience.md 데모 1 weekly-sync 예시

```markdown

- 예: `/weekly-sync` → Slack 메시지 수집 + git 로그 분석 + 보고서 생성
```

**권장 수정**: 21행 뒤에 보충 설명 1줄 추가:

```markdown
**데모 1: Skill 실행 체험**
- Skill = 복잡한 작업을 하나의 명령어로 실행
```

기존 `/weekly-sync` 예시를 유지하면서, 22행(핵심) 앞에 비개발자 친화적 부연 1줄 삽입.
