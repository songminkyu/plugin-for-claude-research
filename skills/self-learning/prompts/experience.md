# Block 1: 체험 — 먼저 느껴보기

## 목표
Claude Code가 어떤 일을 할 수 있는지 3가지 데모를 통해 체험합니다.

## 실행 프로토콜

### Phase A (데모 + 실습 안내)

참조 파일 `references/block1-experience.md`의 EXPLAIN 섹션을 읽고:

**데모 1: Skill 실행 체험**
- 사용자에게 현재 프로젝트의 스킬 목록을 보여주세요
- 예시: `/self-learning`이 어떻게 동작하는지 설명
- "하나의 명령어가 복잡한 작업을 자동으로 처리합니다"

**데모 2: 명확화 패턴 (AskUserQuestion)**
- 모호한 요청 vs 명확한 요청의 차이를 보여주세요
- 예: "보고서 써줘" → Claude가 되묻기 → 더 좋은 결과

**데모 3: 직접 질문하기**
- "Claude Code에서 MCP가 뭐야?" 같은 질문을 해보도록 안내
- Claude Code 자체가 가이드 역할을 할 수 있음을 보여주기

참조 파일의 EXECUTE 섹션을 읽고 실습을 안내하세요:
- 데모 2, 3을 직접 실행해보도록 안내

마무리 멘트:
"👆 위 내용을 직접 실행해보세요. 실행이 끝나면 '완료' 또는 '다음'이라고 입력해주세요."

**여기서 STOP.**

---

### Phase B (퀴즈 + 피드백)

참조 파일의 QUIZ 섹션을 읽고 AskUserQuestion으로 퀴즈:

"세 가지 데모 중 가장 인상적이었던 것은?"
- 옵션 1: "Skill로 복잡한 작업을 한 번에 처리하는 것"
- 옵션 2: "Claude가 되물어서 더 정확한 결과를 내는 것"
- 옵션 3: "Claude에게 직접 물어보면 문서를 안 봐도 되는 것"

→ 정답 없음. 어떤 선택이든 의미를 설명하고 격려.
→ Block 2로 이동 안내.
