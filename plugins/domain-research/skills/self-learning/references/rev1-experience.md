# 레볼루션 1: 체험 — 참조 자료

## EXPLAIN

### Before & After

**Before (지금):**
- 여러 도구를 열어서 수동 작업
- 복사-붙여넣기 반복
- 문서를 찾아 읽고, 직접 정리

**After (Claude Code 이후):**
- 하나의 명령어로 복잡한 작업 자동 처리
- 자연어로 지시 → 결과물 자동 생성
- 되물어서 정확한 결과 보장

### 3가지 데모

**데모 1: Skill 실행 체험**
- Skill = 복잡한 작업을 하나의 명령어로 실행
- 예: `/weekly-sync` → Slack 메시지 수집 + git 로그 분석 + 보고서 생성
- 비개발자 예시: "팀 업무 보고서를 한 명령어로 자동 생성" — Slack이나 git을 몰라도 개념은 동일합니다
- 핵심: 반복 업무를 레시피로 만들어 재사용

**데모 2: 명확화 패턴 (AskUserQuestion)**
- 모호한 요청: "보고서 써줘" → 일반적인 결과
- 명확한 요청 유도: Claude가 "어떤 보고서인가요?", "대상은 누구인가요?" 등 되물음
- 결과: 사용자가 원하는 것에 더 가까운 결과물

**데모 3: 직접 질문하기**
- Claude Code는 자기 자신에 대한 가이드 역할도 함
- "Claude Code에서 MCP가 뭐야?" → 바로 설명
- 문서를 찾아 읽는 대신, 바로 물어보기

### 🖥️ PPTX 프로젝트 적용 예시

Claude Code 첫 체험으로 이것을 해보세요:

"PPTX 5장짜리 팀 소개 슬라이드 만들어줘"

→ Claude가 테마, 내용, 파일 생성까지 한 번에 처리합니다.
→ 이것이 레볼루션 0~6에서 만들어갈 시스템의 최종 결과입니다.
