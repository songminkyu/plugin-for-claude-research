### GUI vs CLI

```
┌─────────────────────────────────┐
│  GUI 도구 (Cursor, CoWork 등)    │
│  → 미리 만들어진 버튼만 사용      │
│  → 기능이 UI에 의해 제한됨        │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│  CLI Claude Code (터미널)        │
│  → 컴퓨터의 모든 기능 접근 가능    │
│  → AI 에이전트가 직접 소프트웨어 제어│
└─────────────────────────────────┘
```

**핵심 차이:**
- GUI: 개발자가 미리 만들어놓은 기능만 사용 가능
- CLI: AI가 컴퓨터를 직접 제어하므로, 가능성이 무한함

### 왜 코딩이 비개발자에게도 중요한가?

- **과거**: 코딩 = 개발자의 전문 기술
- **현재**: 코딩 = AI가 대신 작성, 사용자는 지시만
- **결론**: 코딩이 "개발"이 아닌 "범용 지식 작업 도구"로 전환

Claude Code가 코드를 작성하므로, 사용자는 "무엇을 하고 싶은지"만 말하면 됩니다.

### 🖥️ PPTX 프로젝트 적용 예시

수작업 vs 자동화 비교:

| | 수작업 | Claude Code CLI 자동화 |
|---|---|---|
| 첫 번째 PPTX | 2시간 | 5분 |
| 열 번째 PPTX | 2시간 (매번 동일) | 5분 (명령 하나) |
| 스타일 변경 | 슬라이드 1장씩 수정 | "파란색으로 바꿔줘" |
| 팀 전체 적용 | 각자 반복 | Plugin 1회 설치 |

→ CLI가 없으면 두 번째 PPTX도 수작업입니다.
