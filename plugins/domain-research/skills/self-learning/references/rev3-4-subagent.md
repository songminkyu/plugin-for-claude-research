# 기능 4: Subagent — 참조 자료

## EXPLAIN

📖 공식 문서: https://code.claude.com/docs/ko/sub-agents

### Subagent란?

Subagent = **독립된 공간에서 특정 작업을 전담 처리**하는 별도의 Claude Code 프로세스

### 핵심 원리: 프로세스 격리 + 빈 상태

```
┌──────────────────┐      ┌──────────────────┐
│  메인 대화        │      │  Subagent         │
│  (컨텍스트 유지)   │ ──→ │  (독립 컨텍스트)    │
│                  │      │  작업 처리...       │
│  요약 결과 수신 ← │ ←── │  결과 요약 반환     │
└──────────────────┘      └──────────────────┘
```

- Subagent는 메인 대화의 컨텍스트를 **상속받지 않음**
- 독립된 프로세스에서 작업을 완료하고 **요약만 반환**
- 메인 대화의 컨텍스트 소비를 **방지**

### 비유

```
Subagent = 부하 직원에게 업무 위임
- 메인(상사): "이 폴더 구조 분석해줘"
- Subagent(부하): 독립적으로 분석 후 보고서만 전달
- 결과: 상사의 업무 흐름(컨텍스트)은 방해받지 않음
```

### 언제 사용하나?

| 적합한 경우 | 불필요한 경우 |
|------------|-------------|
| 대량 파일 분석 | 간단한 질문 |
| 폴더 구조 탐색 | 한 줄 수정 |
| 복잡한 코드 리뷰 | 짧은 대화 |

### 🖥️ PPTX 프로젝트 적용 예시

PPTX 생성은 3가지 독립 작업으로 분리하기에 딱 좋습니다:

```
메인 Claude
  ├─→ Subagent A: "슬라이드 테마 설정해줘" (🎨 테마 설정)
  │     결과 반환: theme.json
  │
  ├─→ Subagent B: "이 내용으로 슬라이드 텍스트 작성해줘" (✍️ 내용 수정)
  │     결과 반환: slides-content.md
  │
  └─→ Subagent C: "theme.json + slides-content.md로 PPTX 렌더링해줘" (🖨️ 렌더링)
        결과 반환: output.pptx
```

각 Subagent가 독립적으로 처리 → 메인 대화 컨텍스트 절약 + 병렬 처리 가능
