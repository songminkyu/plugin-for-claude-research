## EXPLAIN

### 학습의 완성: 도구 + 방법론

지금까지 학습한 것:
- **도구**: Claude Code의 7대 핵심 기능
- **실습**: 나만의 Skill 제작

이제 배울 것:
- **방법론**: 체계적 리서치를 통한 심화 학습

### domain-research 스킬 파이프라인

```
┌──────────────────────────────────────────────┐
│  Step 0: 대화형 의도 분석                      │
│  "AI 활용한 업무 자동화가 궁금해요"              │
│  → 맥락 파악, 연구 범위 설정                    │
├──────────────────────────────────────────────┤
│  Step 1: 핵심 질문 생성 (5개)                  │
│  → 검증 가능한 연구 질문 도출                   │
├──────────────────────────────────────────────┤
│  Step 2: 연구 공백 식별                        │
│  → 기존 연구의 한계와 미탐구 영역 발견            │
├──────────────────────────────────────────────┤
│  Step 3: 개별 소스 분석                        │
│  → 논문, 보고서, 사례 심층 분석                  │
├──────────────────────────────────────────────┤
│  Step 4: 다중 소스 통합                        │
│  → 여러 출처의 인사이트 종합                    │
├──────────────────────────────────────────────┤
│  Step 5: 실행 계획 수립                        │
│  → 인사이트를 실행 가능한 액션으로 전환           │
├──────────────────────────────────────────────┤
│  Final: 종합 가이드                            │
│  → 한 페이지 실행 로드맵 생성                   │
└──────────────────────────────────────────────┘
```

### 자기주도 학습 워크플로우

```
1. 관심 주제 선정
   └→ "우리 팀에 Claude Code 도입하기"

2. domain-research로 체계적 탐구
   └→ /research "Claude Code 팀 도입 전략"

3. 학습한 기능으로 자동화
   ├→ Skill: 팀 온보딩 자동화
   ├→ MCP: Slack/Notion 연동
   └→ Hook: 코드 품질 자동 체크

4. 결과물을 Skill + Plugin으로 정리
   └→ 팀 전체에 공유
```

### 🖥️ PPTX 프로젝트 적용 예시

리서치 결과를 PPTX로 자동 변환하는 파이프라인:

```
1. 리서치 시작
   /research "2025년 AI 업무 자동화 트렌드"
   └→ domain-research가 5단계 분석 수행

2. 리서치 결과 → PPTX 구조로 변환
   ✍️ 내용 수정 담당이 리서치 인사이트를 슬라이드 개요로 재구성
   └→ "핵심 발견 5가지" → 5개 슬라이드 초안

3. PPTX 시스템으로 자동 생성
   🎨 테마 적용 → ✍️ 내용 채우기 → 🖨️ 렌더링
   └→ output-AI-trends-2025.pptx 완성

4. 결과물 Plugin으로 배포
   └→ 팀 전체가 같은 리서치-PPTX 파이프라인 사용
```

→ 레볼루션 0~5에서 배운 모든 기능(CLAUDE.md + Skill + MCP + Subagent + Teams + Hook + Plugin)이 이 파이프라인 하나에 통합됩니다.
