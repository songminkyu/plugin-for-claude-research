# 기능 1: CLAUDE.md — 참조 자료

## EXPLAIN

📖 공식 문서: https://code.claude.com/docs/ko/memory

### CLAUDE.md란?

CLAUDE.md는 Claude Code의 **영구 기억 장치**입니다.

- 매 대화 시작 시 **자동으로** 읽혀서 Claude의 행동을 결정
- 프로젝트의 규칙, 선호도, 기술 스택 등을 저장
- 팀 위키처럼 모든 세션에서 일관된 동작 보장

### 핵심 비유

```
CLAUDE.md = 팀 위키 (항상 참조되는 기본 규칙서)
```

### 설정 방법

**1단계: `/init` 명령어**
- 프로젝트 폴더를 분석하여 CLAUDE.md 자동 생성
- 프로젝트 구조, 기술 스택, 빌드 방법 등을 기록

**2단계: `/memory` 명령어**
- 대화 중 발견한 패턴이나 규칙을 영구 저장
- 예: "항상 존댓말 사용", "표 형식으로 출력"

### 로드 구조

```
프로젝트 루트 CLAUDE.md     → 항상 로드
~/.claude/CLAUDE.md        → 전역 설정 (모든 프로젝트에 적용)
하위 폴더 CLAUDE.md        → 해당 폴더 작업 시에만 로드
```

---

### 🖥️ PPTX 프로젝트 적용 예시

CLAUDE.md에 PPTX 스타일 가이드를 정의하면 모든 세션에서 일관된 디자인이 유지됩니다:

```markdown
# PPTX 프로젝트 스타일 가이드

## 디자인 규칙
- 주색상: #2D4A8A (딥블루)
- 폰트: 제목 28pt, 본문 16pt
- 여백: 상하 40px, 좌우 60px

## 역할 담당
- 🎨 테마 설정 담당: 슬라이드 디자인, 색상, 폰트 결정
- ✍️ 내용 수정 담당: 텍스트, 구조, 논리 흐름
- 🖨️ 렌더링 담당: HTML → PPTX 변환, 파일 출력

## 출력 규칙
- 파일명: output-YYYYMMDD.pptx
- 항상 한국어로 슬라이드 작성
```

→ 이 규칙이 CLAUDE.md에 있으면 3명의 담당자 모두 동일한 기준으로 작업합니다.

---

## EXECUTE

1. `/init`을 실행하여 CLAUDE.md를 생성해보세요
2. `/memory "항상 한국어로 응답하기"`처럼 규칙을 저장해보세요
3. 생성된 CLAUDE.md 파일을 읽어보세요
